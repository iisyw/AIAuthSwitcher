# AIAuthSwitcher

AIAuthSwitcher 是一个 VS Code 扩展，用来查看、备份、导入、导出和切换本地 AI 工具授权文件。

当前版本仅支持 Codex，授权文件路径为 `~/.codex/auth.json`。

## 功能

- 展示当前 Codex 账号摘要
- 展示当前账号 Codex 用量
- 备份当前授权到扩展存储目录
- 恢复、删除、勾选导出已保存备份
- 导入单账号 JSON 或多账号合集 JSON 文件
- 导入后自动查询已导入账号的 Codex 用量
- 支持单个查询或批量顺序查询备份账号的 Codex 用量
- 备份列表会按每周窗口已用从低到高排序，并在列表名中显示倒计时
- 通过侧边栏 `+` 入口登录并添加 Codex 账号
- 当前账号 Codex 用量刷新后，如果备份列表里存在同一账号，会同步更新对应备份项的用量缓存
- 切换授权后提示重载 VS Code 窗口
- 即使当前 `~/.codex/auth.json` 已不存在，也可以直接从备份恢复切换

## 使用

1. 在 VS Code 侧边栏打开 `AIAuthSwitcher`。
2. 查看当前账号信息和当前账号 Codex 用量。
3. 使用“操作”区域可以刷新授权、查询当前账号用量、备份当前授权或重载窗口。
4. 在“账号备份列表”区域可以导入授权、批量查询备份账号用量、导出勾选备份，或者对单个备份执行查询、切换、删除。
5. 点击顶部 `+` 按钮，选择 `Codex`，输入邮箱、密码和验证码以添加账号。
6. 切换授权后按提示重载窗口。

授权备份存放在扩展的 VS Code 全局存储目录中。

## 导入说明

- 仅支持从本地 JSON 文件导入。
- 导入文件结构建议与“导出到文件”生成的 JSON 保持一致。
- 支持导入标准授权对象，或包含多个账号对象的合集 JSON。
- 导入完成后，扩展会自动对本次导入的账号查询一遍 Codex 用量。

## 用量显示说明

- 当前账号用量会在视图首次加载时自动查询一次，也可以手动点击“查询 Codex 用量”强制刷新。
- 备份账号支持单个查询和批量顺序查询。
- 备份列表名称格式为 `每周窗口已用 | 距离重置倒计时 | 邮箱`，例如 `32% | 6d 03:14:08 | a@abc.com`。
- 当每周窗口已到或超过重置时间时，列表中显示为 `已至`，不会继续对该项做本地倒计时计算。
- `Free` 套餐不会显示 5 小时窗口相关信息。

## 注意事项

- 扩展会处理本地授权信息，请将导出的备份文件当作凭证保管。
- 扩展不会主动上传授权数据到远程服务。
- 如果目标扩展已经在内存中缓存授权，切换后仍需要重载窗口才会生效。
